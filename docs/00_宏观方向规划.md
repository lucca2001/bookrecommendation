# 读书推荐网站 — 宏观方向规划
> 文档版本：v0.1 | 创建日期：2026-06-21

---

## 一、项目定位

| 维度 | 描述 |
|------|------|
| 核心功能 | 类 MBTI 问卷测评 → 多维用户画像 → 精准图书推荐 |
| 目标用户 | 中文读者（含中译本），覆盖学生、职场人、研究者、休闲读者 |
| 内容广度 | 从专业学术读物到消遣娱乐读物，全品类覆盖 |
| 差异化 | 比豆瓣评分推荐更精准（基于多维测评而非历史行为） |

---

## 二、三大核心模块（待细化）

```
┌─────────────────────────────────────────────┐
│  模块 A：用户测评系统（问卷引擎）              │
│  模块 B：图书数据集（书库）                   │
│  模块 C：推荐匹配引擎（算法层）               │
└─────────────────────────────────────────────┘
```

### 模块 A — 用户测评系统
- 类 MBTI 的多选题/量表题问卷
- 产出用户在多个维度上的「画像向量」
- 维度设计是核心难点（待第二步细化）

### 模块 B — 图书数据集
- 构建覆盖全品类的中文书目数据集
- 每本书需打上多维「标签向量」与分类标签
- 数据来源、字段设计、维护方式待细化

### 模块 C — 推荐匹配引擎
- 将用户画像向量与书目标签向量做匹配
- 输出个性化推荐列表
- 算法方案待细化（规则 / 向量相似度 / 混合）

## 三、宏观技术路线（初步候选）

### 前端
- 交互体验为核心，类似 16Personalities 的沉浸式问卷
- 技术栈候选：React / Vue + 动效库

### 后端
- 问卷逻辑处理、推荐算法服务
- 技术栈候选：Python（FastAPI）/ Node.js

### 数据层
- 图书数据集存储：PostgreSQL / MongoDB
- 推荐向量检索：可选 Elasticsearch / pgvector

---

## 四、两大核心设计挑战

### 挑战 1：测评维度设计（决定推荐精准度的上限）
问卷需要评估哪些维度？候选方向：
- 阅读目的（学习成长 / 消遣娱乐 / 解决问题 / 情感共鸣）
- 知识偏好（理科思维 / 人文思维 / 跨学科）
- 阅读深度（通俗 / 中等 / 学术严肃）
- 情感倾向（积极励志 / 批判反思 / 中性客观）
- 生活场景（学生 / 职场 / 育儿 / 旅行 etc.）
- 时间偏好（历史经典 / 当代 / 最新出版）
- 文化偏好（东方 / 西方 / 本土）
- ……（待系统化设计）

### 挑战 2：图书数据集构建（决定推荐精准度的下限）
- 数据量：需要多少本书才能覆盖全品类？
- 数据来源：豆瓣 / 国家图书馆 / 开放书目 / 人工标注
- 标签体系：如何为每本书打上与测评维度对应的标签？
- 维护机制：如何保持数据集的新鲜度？
