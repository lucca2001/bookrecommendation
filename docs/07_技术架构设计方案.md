> 文档版本：v0.1 | 创建日期：2026-06-21

---

## 一、架构选型原则

| 原则 | 说明 |
|------|------|
| 开发效率优先 | 减少基础设施配置时间，快速出可运行产品 |
| 部署简单 | 一键部署，无需运维服务器 |
| 可扩展 | MVP 架构可平滑演进至正式产品 |
| 前后端分离 | 前端独立部署，后端 API 化，便于后期替换 |

## 三、各层技术选型详解

### 前端：Next.js 14

| 选择原因 | 说明 |
|---------|------|
| React 生态 | 组件化开发，动效库丰富（Framer Motion） |
| App Router | 页面结构清晰，SSG/SSR 混合渲染 |
| API Routes | 后端逻辑直接写在同一项目，无需单独部署后端服务 |
| Vercel 一键部署 | 与 Vercel 原生集成，push 即部署 |
| TypeScript | 类型安全，数据结构清晰 |

**关键依赖库：**

```json
{
  "dependencies": {
    "next": "14.x",
    "react": "18.x",
    "typescript": "5.x",
    "framer-motion": "^11",        // 滚动动效、页面过渡
    "d3": "^7",                    // 雷达图绘制
    "gsap": "^3",                  // Scroll Storytelling 动效
    "@supabase/supabase-js": "^2", // 数据库客户端
    "zustand": "^4",               // 轻量状态管理（问卷答题状态）
    "html2canvas": "^1",           // 分享卡片截图生成
    "tailwindcss": "^3"            // 样式（快速实现暗色设计系统）
  }
}
```

### 后端：Next.js API Routes（Serverless）

MVP 阶段无需独立后端服务，所有 API 通过 Next.js 的 `/app/api/` 目录实现，部署在 Vercel Serverless Functions。

**API 端点设计：**

```
POST /api/recommend
  入参：{ answers: QuizAnswer[] }
  出参：{ bookgrid: BookGrid, books: Book[], radarData: RadarData }
  逻辑：计算用户向量 → 匹配书格 → 向量检索推荐书单

GET  /api/bookgrid/:id
  出参：{ bookgrid: BookGrid }
  逻辑：返回指定书格的完整信息（名称/描述/典型特征）

GET  /api/books?bookgrid_id=xxx
  出参：{ books: Book[] }
  逻辑：返回某书格的推荐书单

GET  /api/books/:id
  出参：{ book: Book }
  逻辑：返回单本书详情
```

**扩展接口预留（后期账号体系）：**
```
POST /api/users          # 用户注册（预留，当前不实现）
POST /api/feedback       # 推荐反馈（预留）
GET  /api/users/:id/history  # 测评历史（预留）
```

### 数据库：Supabase（PostgreSQL + pgvector）

#### 核心表结构

**books 表（图书主表）**
```sql
CREATE TABLE books (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title         TEXT NOT NULL,
  author        TEXT NOT NULL,
  translator    TEXT,
  publisher     TEXT,
  pub_year      INTEGER,
  isbn          TEXT,
  pages         INTEGER,
  cover_url     TEXT,
  description   TEXT,
  douban_id     TEXT,
  douban_score  DECIMAL(3,1),
  douban_rating_count INTEGER,
  
  -- 分类字段
  category_display  TEXT,        -- 对外品类（文学小说等）
  category_sub      TEXT,        -- 二级品类
  clc_code          TEXT,        -- 中图分类号
  tags_douban       TEXT[],      -- 豆瓣标签数组
  
  -- AI标注字段（10轴向量）
  axis_vector       vector(10),  -- pgvector存储，用于相似度检索
  axis_scores       JSONB,       -- 人类可读的10轴分值
  annotation_confidence DECIMAL(3,2),
  annotation_source TEXT,
  needs_review      BOOLEAN DEFAULT false,
  
  -- 推荐统计（可扩展）
  recommendation_count INTEGER DEFAULT 0,
  
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now()
);

-- 向量索引（余弦相似度检索加速）
CREATE INDEX ON books USING ivfflat (axis_vector vector_cosine_ops)
  WITH (lists = 100);
```

**bookgrids 表（书格定义表）**
```sql
CREATE TABLE bookgrids (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  slug            TEXT UNIQUE NOT NULL,  -- e.g. "shunzhe-shu-zhao-shu"
  name            TEXT NOT NULL,         -- 「顺着书找书型」
  naming_type     TEXT,                  -- behavior / identity
  description     TEXT NOT NULL,         -- 人格描述文案（有梗）
  short_desc      TEXT,                  -- 一句话描述
  traits          TEXT[],                -- 3条典型特征
  
  -- 原型向量（用于书格匹配）
  prototype_vector  vector(10),
  
  -- 分享卡片文案
  share_tagline   TEXT,
  
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

**quiz_questions 表（问卷题目表）**
```sql
CREATE TABLE quiz_questions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_index   INTEGER NOT NULL,
  question_type TEXT NOT NULL,   -- choice / slider / image / fun
  question_text TEXT NOT NULL,
  options       JSONB,           -- 选项及其轴分值映射
  axis_mapping  JSONB,           -- { axis_id: weight } 该题影响哪些轴
  is_active     BOOLEAN DEFAULT true,
  
  created_at    TIMESTAMPTZ DEFAULT now()
);
```

**bookgrid_books 表（书格-图书关联表）**
```sql
CREATE TABLE bookgrid_books (
  bookgrid_id   UUID REFERENCES bookgrids(id),
  book_id       UUID REFERENCES books(id),
  is_surprise   BOOLEAN DEFAULT false,   -- 是否为「意外推荐」
  rank          INTEGER,                  -- 推荐优先级
  reason        TEXT,                     -- 有梗的推荐理由
  
  PRIMARY KEY (bookgrid_id, book_id)
);
```

**quiz_sessions 表（测评会话，可扩展为用户历史）**
```sql
CREATE TABLE quiz_sessions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_token   TEXT UNIQUE,           -- 无账号时用临时token
  user_id         UUID,                  -- 预留，账号体系启用后填充
  answers         JSONB NOT NULL,        -- 原始答题记录
  user_vector     vector(10),            -- 计算出的用户向量
  bookgrid_id     UUID REFERENCES bookgrids(id),
  recommended_books UUID[],              -- 最终推荐的书单
  
  created_at      TIMESTAMPTZ DEFAULT now()
);
```

## 五、部署方案

```
开发环境
  └── 本地 next dev + Supabase 本地实例（supabase start）

预览环境
  └── Vercel Preview（每个 PR 自动生成预览链接）

生产环境
  └── Vercel Production（main 分支 push 自动部署）
  └── Supabase 云端（免费套餐：500MB存储，50000月活）

域名
  └── Vercel 免费域名（xxx.vercel.app）→ 后期绑定自定义域名
```
