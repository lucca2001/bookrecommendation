# 推荐匹配算法方案设计
> 文档版本：v0.1 | 创建日期：2026-06-21

---

## 一、算法目标

将用户测评产生的「10维画像向量」与书库中每本书的「10维标注向量」做匹配，
输出 3–5 本推荐书单，其中：
- **2–3 本**：高精准匹配（强相关）
- **1–2 本**：「适度意外」推荐（高质量但用户可能没想到）

---

## 二、整体算法架构

```
用户完成问卷
    ↓
Step 1：生成用户画像向量 (10维, 每维 0–100)
    ↓
Step 2：书格匹配 → 确定用户属于哪种「书格」
    ↓
Step 3：精准推荐候选池筛选（向量相似度计算）
    ↓
Step 4：意外推荐候选池筛选（偏移探索策略）
    ↓
Step 5：综合排序 + 去重 → 输出最终 3–5 本
```

---

## 三、Step 1：用户画像向量生成

问卷每道题对应 1–3 个维度轴，答案映射为轴上的分值增量。

### 示例题目映射

```
题目：「周末你更想做什么？」
  A. 读一本感觉能学到东西的书   → 轴1(探索) -15, 轴4(实用) -10
  B. 读一本让自己完全沉浸进去   → 轴1(情感) +15, 轴4(逃逸) +10
  C. 随手翻翻，不想太费脑子     → 轴7(轻盈) +15, 轴3(涉猎) +10
  D. 其实不太想读，但会强迫自己  → 轴9(规律) -10

题目：「你上一本读完的书是因为什么选的？」
  A. 豆瓣/榜单推荐              → 轴10(分享) +10
  B. 朋友推荐，想和他聊          → 轴10(分享) +20
  C. 看到书名/封面就买了         → 轴2(感性) +15
  D. 研究某个问题时找到的         → 轴1(探索) -20, 轴2(逻辑) -15
```

所有题目答案分值累加 → 归一化到 0–100 → 得到用户 10 维向量。

## 四、Step 2：书格匹配算法

### 方法：加权欧氏距离最近邻

预先定义 16 种书格的「原型向量」（每种书格在 10 轴上的典型分值）。

```python
# 伪代码
def match_bookgrid(user_vector):
    min_distance = float('inf')
    matched_grid = None
    
    for grid in BOOKGRID_PROTOTYPES:  # 16种书格原型
        distance = weighted_euclidean(
            user_vector, 
            grid.prototype_vector,
            weights=AXIS_WEIGHTS  # 不同轴的权重不同
        )
        if distance < min_distance:
            min_distance = distance
            matched_grid = grid
    
    return matched_grid
```

### 轴权重设计（初稿）

| 维度轴 | 权重 | 原因 |
|--------|------|------|
| 轴1 阅读驱动力 | 1.5 | 最能区分读者根本类型 |
| 轴2 思维偏好 | 1.2 | 影响内容形式偏好 |
| 轴3 阅读深度 | 1.3 | 直接影响书籍难度匹配 |
| 轴4 内容取向 | 1.4 | 实用 vs 消遣是核心分叉 |
| 轴5 时空偏好 | 1.0 | 中等权重 |
| 轴6 文化圈层 | 1.0 | 中等权重 |
| 轴7 叙事口味 | 1.2 | 影响阅读体验满意度 |
| 轴8 世界观取向 | 1.1 | 影响内容价值观匹配 |
| 轴9 阅读节奏 | 0.8 | 辅助维度 |
| 轴10 社交倾向 | 0.7 | 辅助维度 |

## 五、Step 3：精准推荐候选池筛选

### 方法：余弦相似度检索

在书库中找与用户向量余弦相似度最高的书，筛选出 Top 20 候选。

```python
def get_precise_candidates(user_vector, book_db, top_k=20):
    scores = []
    for book in book_db:
        similarity = cosine_similarity(user_vector, book.axis_scores)
        scores.append((book, similarity))
    
    # 过滤低置信度书目
    scores = [(b, s) for b, s in scores if b.annotation_confidence > 0.75]
    
    return sorted(scores, key=lambda x: -x[1])[:top_k]
```

从 Top 20 中最终选 **2–3 本**，选择规则：
1. 相似度最高的 1 本必选
2. 品类不重复（避免推 3 本同类书）
3. 优先选「豆瓣评分较高 + 该书格历史推荐反馈好」的

## 六、Step 4：「适度意外」推荐策略

### 核心思路：向量偏移探索

在用户向量基础上，**向某 1–2 个次强维度方向偏移**，找到「用户没有明确表达但潜在契合」的书。

```python
def get_surprise_candidates(user_vector, book_db):
    # 找出用户得分在 40–60 之间的「模糊维度」（最有探索空间）
    ambiguous_axes = [i for i, v in enumerate(user_vector) if 35 < v < 65]
    
    surprise_books = []
    for axis in ambiguous_axes[:2]:  # 最多探索2个维度
        # 向该维度的两端各偏移 20 分，看看有没有高质量书
        shifted_vector = user_vector.copy()
        shifted_vector[axis] += 25  # 向一端偏移
        
        candidates = cosine_similarity_search(shifted_vector, book_db, top_k=10)
        # 筛选高质量（豆瓣评分 > 8.0）且与精准推荐不重复的书
        surprise_books.extend([b for b, s in candidates if b.douban_score > 8.0])
    
    return surprise_books[:2]  # 最多2本意外推荐
```

### 意外推荐的展示方式
在结果页单独标注：
> 💡 **你可能没想到，但很适合你：**《书名》
> *理由：你在「XX」维度有隐藏倾向，这本书可能会让你惊喜*

## 七、Step 5：综合排序与最终输出

```python
def final_recommendation(precise_books, surprise_books, user_bookgrid):
    result = []
    
    # 精准推荐：2–3本，按相似度排序
    result.extend(precise_books[:3])
    
    # 意外推荐：1–2本，附带理由标签
    for book in surprise_books[:2]:
        book.is_surprise = True
        book.surprise_reason = generate_reason(book, user_bookgrid)
        result.append(book)
    
    # 去重
    result = deduplicate(result)
    
    # 最终输出 3–5 本
    return result[:5]
```

## 八、冷启动与持续优化

### 冷启动（MVP阶段无用户数据）
- 完全依赖向量相似度 + 豆瓣评分加权
- 每种书格预先人工审核推荐结果，确保初版质量

### 持续优化（上线后）
```
用户行为信号 → 反馈权重调整
  ├── 「推荐很准」反馈   → 提升该书在同书格的推荐权重
  ├── 「推荐不准」反馈   → 降低权重 + 触发该书重新标注
  ├── 点击查看详情       → 弱正向信号
  └── 分享结果页        → 强正向信号（书格命中感强）
```

### 未来可扩展方向（留接口）
- 协同过滤：相似用户的阅读偏好互补（需账号体系）
- 序列推荐：「读完这本，下一本读什么」
- 情境推荐：根据当前心情/季节调整权重
