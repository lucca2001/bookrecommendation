## 一、设计风格定位

### 参考标杆：The Pudding 风格
核心 DNA：
- **Scroll-driven Storytelling**：用户向下滚动推进叙事，每一屏是一个独立的「章节」
- **大字冲击 + 克制排版**：超大号文字作为视觉锚点，大量留白，文字密度低
- **动效有意义**：动画不是装饰，是叙事的一部分（滚动触发、状态转换）
- **黑白为主色调**：偶尔用 1–2 个强调色制造视觉焦点
- **文案有个性**：不说废话，每一句都有态度

### 应用到本项目的调整
- 背景色：#0f0f0f（近黑）
- 主文字：#f5f0e8（暖白/米白，书页感）
- 强调色：#e8c547（暖金黄，书页泛黄的意象）
- 字体：标题用衬线体（Noto Serif SC / 思源宋体），正文用无衬线体
- 整体气质：「有点严肃但其实很好玩」

## 二、整体页面结构

```
┌─────────────────────────────────┐
│  Section 0：Landing（入场）       │  全屏，1个大标题 + 开始按钮
├─────────────────────────────────┤
│  Section 1：引子（钩子）          │  滚动叙事，3–4屏，建立「书格」概念
├─────────────────────────────────┤
│  Section 2：问卷（核心交互）      │  20–30题，每题全屏，滚动推进
├─────────────────────────────────┤
│  Section 3：计算过渡页            │  动效loading，制造期待感
├─────────────────────────────────┤
│  Section 4：结果页（书格揭晓）    │  大字揭晓 + 人格描述 + 书单
├─────────────────────────────────┤
│  Section 5：分享 & 探索其他书格   │  社交分享 + 看其他16种书格
└─────────────────────────────────┘
```

### Section 1：引子（Scroll Storytelling，4屏）

**屏1**（滚动进入）：
```
每个人读书的方式，

```
*文字从模糊到清晰，向上浮现*

**屏2**：
```

*右侧出现一摞书的线条插画，书名故意模糊*

**屏4**：

```
*强调色文字，过渡到问卷*

### Section 2：问卷设计（每题全屏）

#### 题目类型 A：二选一情景题
```
┌──────────────────────────────────────────────┐
│                                              │
│  第 3 题 / 20                                │  进度条（顶部细线）
│                                              │
│                                              │
│    周末，你拿起一本书。                        │
│    你更可能在想：                             │
│                                              │
│  ┌─────────────────┐  ┌─────────────────┐   │
│  │                 │  │                 │   │
│  │  「这本书能让   │  │  「我只是想     │   │
│  │   我学到什么？」 │  │   消失一下。」  │   │
│  │                 │  │                 │   │
│  └─────────────────┘  └─────────────────┘   │
│                                              │
│     ← 上一题              自动进入下一题 →    │
│                                              │
└──────────────────────────────────────────────┘
```
- 选中后：卡片变色（强调色边框），0.3s 后自动进入下一题
- 不设「下一步」按钮，选完即走，节奏更流畅

#### 题目类型 B：滑块量表题
```
┌──────────────────────────────────────────────┐
│                                              │
│  第 7 题 / 20                                │
│                                              │
│    你上一本买的书，有没有读完？               │
│                                              │
│  还没开始读    ●─────────────────    读了很多遍 │
│                                              │
│  （拖动滑块）                                │
│                                              │
└──────────────────────────────────────────────┘
```

#### 题目类型 C：多图选择题（封面直觉）
```
┌──────────────────────────────────────────────┐
│                                              │
│  第 12 题 / 20                               │
│                                              │
│    直觉反应：哪个封面最吸引你？               │
│                                              │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐    │
│  │封面A │  │封面B │  │封面C │  │封面D │    │  4个真实书籍封面
│  └──────┘  └──────┘  └──────┘  └──────┘    │
│                                              │
└──────────────────────────────────────────────┘
```
- 4 个封面代表 4 种不同风格（严肃/轻松/东方/西方）
- 这种题无需用户思考，靠直觉，最能暴露真实偏好

#### 题目类型 D：有梗的画风题（增加趣味）
```
┌──────────────────────────────────────────────┐
│                                              │
│  第 15 题 / 20                               │
│                                              │
│    诚实回答：你现在同时在「读」几本书？        │
│                                              │
│      ○  1本（我是正常人）                    │
│      ○  2–3本（还好还好）                    │
│      ○  4–6本（有点失控了）                  │
│      ○  7本以上（书架是我的精神支柱）         │
│                                              │
└──────────────────────────────────────────────┘
```
选项本身就是梗，用户会笑然后选。

### Section 3：计算过渡页

```
正在分析你的书格……

[动效：10个维度轴依次亮起，像仪表盘校准]

「你的阅读基因正在解码中」
```
持续 2.5–3 秒，不能太长，制造期待感而不是焦虑。

## 四、交互细节规范

| 交互 | 规范 |
|------|------|
| 页面进入动效 | 文字从下向上浮现，opacity 0→1，duration 0.6s，ease-out |
| 问卷选项 hover | 边框从灰色变金色，背景轻微变亮，duration 0.15s |
| 选项选中 | 填充金色背景，0.3s 后自动进入下一题 |
| 滚动触发 | IntersectionObserver，进入视口 30% 时触发动画 |
| 进度条 | 顶部 2px 细线，随题目推进，颜色渐变 |
| 雷达图绘制 | SVG 动态绘制，每条轴延迟 0.1s 依次亮起 |
| 书格名称揭晓 | transform scale 0.3→1，同时 opacity 0→1 |
| 移动端适配 | 全部为单列布局，字体等比缩小，touch 替代 hover |

## 五、技术实现要点

| 功能 | 技术方案 |
|------|---------|
| Scroll Storytelling | Intersection Observer API / GSAP ScrollTrigger |
| 雷达图 | D3.js 或 Chart.js（SVG动画） |
| 问卷状态管理 | React Context / Zustand（轻量） |
| 分享卡片生成 | html2canvas → 导出图片 |
| 路由 | React Router（结果页有独立URL，可分享） |
| 字体 | 思源宋体（标题）+ 思源黑体（正文），CDN 加载 |
