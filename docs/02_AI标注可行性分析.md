# AI 辅助图书标注可行性分析
> 文档版本：v0.1 | 创建日期：2026-06-21

---

## 一、问题定义

我们需要为 2000–10000 本中文图书打上「与测评维度对应的多维标签向量」。
人工逐本阅读不可行，核心问题是：**AI 能否替代人工完成高质量的书目标注？**

---

## 二、AI 标注的信息来源

AI 标注不依赖「读完全书」，而是依赖以下已存在的结构化信息：

| 信息来源 | 可获取内容 | 质量评估 |
|----------|-----------|---------|
| 豆瓣图书页 | 简介、作者、出版社、读者标签、评分、短评 | ★★★★☆ 覆盖广，短评含读者真实感受 |
| 出版社官方简介 | 内容描述、目标读者、推荐语 | ★★★☆☆ 有营销色彩但结构化好 |
| 亚马逊/当当 | 编辑推荐、读者评论 | ★★★☆☆ 补充视角 |
| 国家图书馆 / 中图分类号 | 标准学科分类（CLC） | ★★★★★ 权威分类，适合一级品类 |
| 维基百科 / 百度百科 | 作者背景、作品背景、主题 | ★★★☆☆ 经典书目质量高 |
| OpenLibrary / WorldCat | 国际标准分类、主题词 | ★★★☆☆ 英文资源，需映射 |

**结论**：AI 标注的本质是「基于书目元数据的语义推断」，不需要读全书。

---

## 三、AI 标注可行的维度 vs 困难的维度

### ✅ AI 标注准确率高的维度

| 维度 | 原因 |
|------|------|
| 学科品类（一级/二级） | 中图分类号直接对应，几乎 100% 准确 |
| 阅读难度（通俗/中等/学术） | 出版社定位 + 豆瓣标签 + 评论语言风格可推断 |
| 文化背景（东方/西方/本土） | 作者国籍 + 出版信息直接判断 |
| 时代背景（经典/当代） | 出版年份直接判断 |
| 阅读时长估算 | 页数直接计算 |
| 核心主题词（3–5个） | 简介 + 豆瓣短评用 LLM 抽取，准确率高 |

### ⚠️ AI 标注需要人工复核的维度

| 维度 | 难点 |
|------|------|
| 情感基调（治愈/烧脑/压抑/励志） | 简介常有主观滤镜，需短评印证 |
| 阅读目的（成长/娱乐/解惑/共鸣） | 同一本书目的多元，需权重分配 |
| 适合人群（细粒度场景） | 需要更多上下文推断 |
| 争议性/思想倾向 | AI 标注有误判风险，敏感维度需人工 |

### ❌ AI 标注准确率低的维度（需人工或跳过）

| 维度 | 原因 |
|------|------|
| 叙事节奏（快/慢）| 需要真实阅读体验，简介无法判断 |
| 作者风格细腻度 | 主观性极强 |

## 四、AI 标注工作流设计

```
Step 1：批量获取元数据
  └─ 来源：豆瓣 API / 爬虫 + 国家图书馆数据
  └─ 字段：书名、作者、简介、CLC分类、豆瓣标签、豆瓣短评Top20

Step 2：LLM 结构化标注（批量调用）
  └─ 输入：上述元数据打包成 Prompt
  └─ 输出：JSON 格式的多维标签向量
  └─ 模型：GPT-4o / Claude 3.5 / 本地模型均可

Step 3：置信度过滤
  └─ 对每个标签输出置信度分数
  └─ 高置信度（>0.85）→ 直接入库
  └─ 中置信度（0.6–0.85）→ 标记待复核
  └─ 低置信度（<0.6）→ 人工标注队列

Step 4：抽样人工审核（质量控制）
  └─ 每个品类抽 5–10% 人工复核
  └─ 建立「标注错误样本库」用于 Prompt 优化

Step 5：持续迭代
  └─ 用户反馈（「这本书推荐不准」）反哺标注修正
```

## 五、成本估算

| 方案 | 处理 5000 本书的成本估算 | 时间 |
|------|------------------------|------|
| GPT-4o（每本约 2000 token） | 约 $50–100 美元 | 数小时（批处理） |
| Claude 3.5 Haiku（更经济） | 约 $10–30 美元 | 数小时 |
| 本地部署开源模型（Qwen/DeepSeek） | 仅算力成本，接近 $0 | 较慢但可接受 |
| 纯人工 | 每本 10 分钟 × 5000 本 ≈ 833 小时 | 完全不可行 |

**结论：AI 标注成本极低，是唯一可行路径。**

## 六、标注质量预期

| 标注类型 | 预期准确率 |
|----------|-----------|
| 一级品类分类 | 95%+ |
| 阅读难度 | 85–90% |
| 核心主题词 | 80–90% |
| 情感基调 | 70–80%（需复核） |
| 适合人群 | 75–85% |

整体来看，**结构化维度准确率高，主观性维度准确率中等**，通过抽样复核和用户反馈可持续优化。

---

## 七、结论与建议

> **AI 标注完全可行**，是本项目数据集构建的核心路径。
> 建议采用「LLM 批量标注 + 置信度过滤 + 抽样人工复核 + 用户反馈迭代」的四层质量保障体系。
> 初版可先用 1000 本精标数据验证推荐效果，再扩展至 5000–10000 本。
