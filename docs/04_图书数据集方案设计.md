## 二、品类体系设计（双层结构）

### 设计原则
- **对外展示层**：口语化分类，用户看得懂，贴近豆瓣/日常认知
- **内部标注层**：中图分类法（CLC）标准代码，权威且便于数据互通
- 两层之间建立映射表，互相转换

## 三、数据来源策略（H1 + H2 混合）

### 来源一：豆瓣高分榜（主力来源，约占 60%）
- 筛选条件：评分 ≥ 8.0，评价人数 ≥ 1000
- 覆盖范围：豆瓣各品类排行榜 Top 500
- 预计入库：约 2000–3000 本
- 优势：质量有保证，元数据丰富（简介/短评/标签齐全）

### 来源二：畅销榜单（大众覆盖，约占 25%）
- 来源：当当网畅销榜、京东图书榜、微信读书热榜
- 筛选条件：近 3 年榜单 + 豆瓣评分 ≥ 7.0
- 预计入库：约 500–1000 本
- 优势：覆盖大众读者实际在读的书，避免数据集过于「文艺精英化」

### 来源三：冷门高分补充（小众覆盖，约占 15%）
- 豆瓣评分 ≥ 8.5，但评价人数 500–5000（小众但高质量）
- 每个品类手动补充 20–50 本代表性冷门好书
- 预计入库：约 500–800 本
- 优势：提升推荐惊喜感，区别于大众推荐平台

## 四、每本书的数据字段设计

### 基础字段（必填）

```json
{
  "book_id": "唯一ID",
  "title": "书名",
  "author": "作者",
  "translator": "译者（中译本填写）",
  "publisher": "出版社",
  "pub_year": 2020,
  "isbn": "978-xxx",
  "pages": 320,
  "language_origin": "zh / en / ja / fr ...",
  "cover_url": "封面图片URL",
  "douban_id": "豆瓣书目ID",
  "douban_score": 8.5,
  "douban_rating_count": 12000,
  "description": "简介（200字以内）"
}
```

### 分类字段

```json
{
  "category_display": "文学小说",
  "category_sub": "外国文学",
  "clc_code": "I",
  "tags_douban": ["外国文学", "小说", "意识流"],
  "tags_custom": ["诺奖", "经典", "难读但值得"]
}
```

### AI 标注字段（核心）

```json
{
  "axis_scores": {
    "drive": 75,        // 轴1：探索(0) ←→ 情感(100)
    "thinking": 65,     // 轴2：逻辑(0) ←→ 感性(100)
    "depth": 20,        // 轴3：精读(0) ←→ 涉猎(100)
    "purpose": 80,      // 轴4：实用(0) ←→ 逃逸(100)
    "time": 70,         // 轴5：现代(0) ←→ 历史(100)
    "culture": 85,      // 轴6：东方(0) ←→ 西方(100)
    "tone": 15,         // 轴7：严肃(0) ←→ 轻盈(100)
    "worldview": 25,    // 轴8：批判(0) ←→ 建构(100)
    "pace": 30,         // 轴9：规律(0) ←→ 随性(100)
    "social": 40        // 轴10：独享(0) ←→ 分享(100)
  },
  "annotation_confidence": 0.87,   // AI标注置信度
  "annotation_source": "llm_v1",   // 标注版本
  "needs_review": false             // 是否需要人工复核
}
```

### 扩展字段（预留，后期账号体系使用）

```json
{
  "user_ratings": [],       // 用户评分（预留）
  "user_reviews": [],       // 用户书评（预留）
  "recommendation_count": 0 // 被推荐次数统计
```

## 五、数据获取技术方案

### Step 1：元数据采集
```
豆瓣图书 API / 爬虫
  → 获取：书名、作者、简介、评分、短评 Top20、豆瓣标签
  → 字段：基础字段 + 分类字段（豆瓣标签部分）

国家图书馆数据 / ISBN 查询 API
  → 获取：CLC分类号、标准书目信息
  → 字段：clc_code
```

### Step 2：AI 批量标注
```
输入 Prompt 结构：
  书名 + 作者 + 简介 + 豆瓣短评Top10 + 豆瓣标签
  → 要求 LLM 输出 10 轴的 0-100 分值 + 置信度

模型选择：
  - 主力：DeepSeek / Qwen（成本低，中文理解好）
  - 复核：Claude / GPT-4o（置信度低的样本）

批处理：
  - 每批 50 本，并行调用
  - 输出 JSON 格式，直接入库
```

### Step 3：质量控制
```
置信度 > 0.85  → 直接入库
置信度 0.6–0.85 → 标记 needs_review = true
置信度 < 0.6   → 人工标注队列

每品类抽 5% 人工复核，建立错误样本库
用户反馈「推荐不准」→ 触发该书重新标注
```
